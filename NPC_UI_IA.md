# NPC 后台页面 IA、交互状态与逐页验收单
### 配套：`NPC_Case_ContentCompany.md` 硬范围 + `NPC_Arch_Layered.md` 领域模型

> 本文定位：在架构设计确定数据与 API 之后，给出后台 Dashboard 与前台回放的页面信息架构、核心交互状态、逐页验收标准。工程实现需以本文为 UI 交付验收依据。

---

## 0. 设计总则

### 0.1 后台设计目标

1. **把七层模型从抽象概念变成可浏览、可追踪、可解释的组织资产。**
2. **任何状态变化都能回答：哪天变了？什么变了？谁导致的？触发了什么结果？**
3. **前后台形成闭环：后台看因果，前台看结果，双向可跳转。**
4. **不追求功能堆砌，优先保证每个页面有清晰的“时间 + 关系 + 行动”三维信息。**

### 0.2 核心交互范型

后台统一使用三种交互范型，贯穿所有页面：

| 范型 | 说明 | 出现位置 |
|------|------|---------|
| **Day 选择器** | 顶部全局选择当前查看的模拟日 | 所有后台页、前台回放 |
| **三件套 Tab** | 快照 / 变更 / 来源事件，按层统一 | 七层每个独立页 |
| **事件→变更→实体 穿透** | 任何变更可追溯到来源事件；任何事件可展开影响 | 事件流、各层页、前台抽屉 |

### 0.3 视觉层级原则

- **时间轴是隐形主轴**：所有页面都默认以 Day 为排序依据。
- **变化 > 静态**：优先展示“今天新增/更新/推翻/触发了什么”，静态定义折叠或置底。
- **关系可视化**：组织图、记忆引用链、规则触发链、资金流向尽量用图/链呈现，不只用表格。
- **异常突出**：规则违规、资源红线、生长信号、新员工观察期使用强色块和徽标。

## 1. 全局信息架构

### 1.1 后台导航结构

左侧导航分四大组，按使用频率和认知顺序排列：

```
AGI DAILY 后台
│
├─ 运行中心
│   ├─ 公司总览（Dashboard）
│   ├─ 工作事件流
│   └─ 日结报告
│
├─ 七层资产
│   ├─ 使命层
│   ├─ 能力层（工具库）
│   ├─ 记忆层
│   ├─ 组织层
│   ├─ 规则层
│   ├─ 资源织网
│   └─ 生长协议
│
├─ 治理
│   ├─ 董事会记录
│   └─ Growth Protocol 控制台
│
└─ 前台
    └─ 日报回放
```

### 1.2 全局顶部栏

顶部固定栏从左到右：

```
[Logo] AGI Daily 后台    [Day 选择器 ▼]    [运行状态 ● 运行中 Day 12]    [▶ / ⏸ / ⏭ 控制]    [🔔 未读事件 3]
```

- **Day 选择器**：下拉显示所有已完成天，董事会日带 ★ 标记，当前选中天高亮。
- **运行状态**：
  - 🟢 运行中（自动推进中）
  - 🟡 暂停（当前天跑完停止）
  - ⚪ 空闲（等待启动）
  - 🔴 董事会等待决策（workflow suspended）
- **控制按钮**：
  - ▶ 开始 / ⏸ 暂停 / ⏭ 推进一天
  - 董事会等待态显示“打开董事会面板”按钮。

### 1.3 全局状态规则

| 状态 | 触发条件 | 全局表现 |
|------|---------|---------|
| 运行中 | SimClock 在自动推进 | 顶部状态绿点，控制按钮显示 ⏸ |
| 暂停 | 用户点击暂停 | 黄点，当前天完成后停止，提示“当天结束后暂停” |
| 空闲 | 无模拟运行 | 灰点，显示 ▶ 开始 |
| 董事会等待 | 每 7 天 workflow suspend | 红点闪烁，导航“董事会”入口高亮，顶部弹出决策面板 |
| 资源红线 | Capital < 30 天成本或 Rep 连续 7 天下降 | 顶部红色横幅告警，跳转资源织网 |
| 生长信号 | 触发扩张/收缩阈值 | 顶部黄色提示条，显示信号摘要 |

| D-6 | 事件流摘要 | 展示当天前 20 条事件，支持滚动加载和穿透跳转 |
| D-7 | 员工贡献 | 展示当天各 Agent 工作量、决策数、成本 |

### 2.2 工作事件流 /events

**页面定位**：还原当天 Agent 协作过程，是全系统的“因果录像带”。

**布局**：

```
┌─────────────────────────────────────────────────────────────────────┐
│ 工作事件流 · Day 12                    [筛选 ▼] [按类型 ▼] [搜索]  │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  [时间轴]                                                            │
│                                                                     │
│  06:00  ━━━ [总编] thinking  今日议程制定...                        │
│         ━━━ [总编] decision  今日重点：大模型推理效率               │
│  06:30  ━━━ [编辑] tool_call  queryArticles({tag:'inference'})      │
│  07:00  ━━━ [编辑→总编] message  提交 10 篇稿件，质量分均值 7.8    │
│  08:00  ━━━ [总编] decision  ✅ 批准发布                            │
│         ━━━ [总编] rule_trigger  HARD_SOURCE_URL_REQUIRED passed    │
│  09:00  ━━━ [编辑] tool_call  publishArticle(...)                   │
│  17:00  ━━━ [总编] settlement  Day 12 日结完成                      │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

**事件卡片设计**：

| 事件类型 | 视觉样式 | 默认状态 |
|---------|---------|---------|
| thinking | 灰色小字气泡 | 折叠，可展开 |
| message | 双栏对话卡片，左发右收 | 展开 |
| tool_call | 代码风格卡片，深底色 | 展开，显示工具名+入参 |
| tool_result | 代码风格卡片，浅底色 | 折叠，展开显示返回值 |
| decision | 蓝色边框卡片，左侧蓝色竖线 | 展开，高亮 |
| board | 黄色全宽卡片，带董事会徽标 | 展开，高亮 |
| rule_trigger | 规则色边框，图标 ⚖️ | 展开 |
| org_change | 组织色边框，图标 🏢 | 展开 |
| memory_write | 记忆色边框，图标 🧠 | 展开 |
| growth_signal | 生长色边框，图标 🌱 | 展开 + 顶部提示 |

**交互状态**：

- 点击事件卡片：
  - 展开“影响面板”：列出该事件触发的所有 `<layer>_changes`、`tool_calls`、`memory_writes`、`rule_executions`、`settlement_drivers`
  - 每条影响可点击跳转到对应层详情
- 筛选器：按事件类型、按 Agent、按影响层级
- 时间轴缩放：全部 / 仅决策 / 仅工具 / 仅异常
- 董事会日：时间轴顶部固定显示董事会输入面板

**验收单**：

| # | 验收项 | 验收标准 |
|---|--------|---------|
| E-1 | 事件列表 | 按 seq 顺序展示当天所有事件，不丢事件 |
| E-2 | 事件卡片 | 不同类型有明确视觉区分，thinking 默认折叠 |
| E-3 | 展开影响 | 点击事件可看到它导致的所有变更，变更可跳转 |
| E-4 | 筛选 | 支持按类型/Agent/层级筛选 |
| E-5 | 董事会日 | 董事会日顶部固定董事会输入面板 |
| E-6 | 实时推送 | 运行中自动追加新事件，不刷新页面 |

| P-3 | 指标顶栏 | 显示当日 DAU/Reputation/Capital，来自资源织网 |
| P-4 | 七层叠加 | 可展开今日七层变更摘要 |
| P-5 | 幕后抽屉 | 文章卡片可展开幕后，展示记忆引用和来源事件 |
| P-6 | 前后台跳转 | 来源事件可跳转到工作事件流 |

## 3. 跨页面通用组件

### 3.1 Day 选择器

- 下拉列表展示所有已完成 Day
- 当前选中 Day 高亮
- 董事会日带 ★
- 有异常事件 Day 带小红点
- 支持键盘左右切换

### 3.2 三件套 Tab

每个层页面统一提供三个 Tab：

- **快照**：当日该层整体状态（适合了解“现在什么样”）
- **变更**：当日该层发生了哪些变化（适合了解“今天变了什么”）
- **来源事件**：当日影响该层的 work_events（适合了解“谁导致的”）

Tab 切换不改变 URL，但可收藏当前状态。

### 3.3 事件影响面板

在事件流或来源事件 Tab 中点击事件后展开：

- 触发哪些层变更
- 调用哪些工具
- 写入/读取哪些记忆
- 触发哪些规则
- 影响哪些资源指标

每个影响项都是可点击链接。

### 3.4 实体引用链

在记忆、规则、资源、员工等页面，使用统一引用链组件：

```
记忆 A → 被文章 X 引用 → 在 Day 12 选题决策中被读取
```

鼠标 hover 显示摘要，点击跳转。

## 4. 页面状态与异常处理

### 4.1 空状态

| 场景 | 展示内容 |
|------|---------|
| 当前 Day 无事件 | “Day N 尚未开始运行” + 启动按钮 |
| 某层当日无变更 | “今日该层无变更” + 引导查看快照 |
| 无规则违规 | “今日无违规” + 显示通过次数 |
| 无生长信号 | “当前无生长/收缩信号” |
| 新员工观察期结束 | 显示最终结果（graduated/failed） |

### 4.2 加载状态

- 页面级：骨架屏
- 卡片级：脉冲占位
- 事件流：底部“加载更多”
- 实时推送：新事件从顶部滑入

### 4.3 错误状态

- API 失败： toast 提示 + 重试按钮
- 数据不一致：黄色提示“该数据可能尚未同步”
- 权限不足：当前员工角色无此页面访问权限（后续迭代）

## 5. 响应式策略

后台主要面向桌面端，但需保证最小宽度 1280px 正常展示，1024px 可横向滚动。

- 左侧导航固定 240px
- 主内容区最小 1000px
- 七层卡网格：≥1440px 7 列，1280px 4 列，1024px 2 列
- 事件流时间轴：窄屏时折叠 thinking/tool_result
