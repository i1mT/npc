# NPC 具象案例：AGI Daily
### 一家由 Agent 运营的 AI 资讯媒体公司

---

## 基本定位

**品类**：AI/科技领域中文日报（面向中文读者的全球 AI 信息聚合与再创作）  
**原材料**：agidaily.db — 全网爬取的 AI 相关文章（唯一的真实数据）  
**运行模式**：**全模拟推演**——资本、用户、广告、外部平台均为模拟环境提供，按天推进  
**北极星指标**：日活读者数（DAU）× 广告 CPM

---

## 七层具象映射

---

### 1. 使命层 Mission

```
我们存在是为了：让中文读者用最少时间，读懂全球 AI 最重要的进展。
```

| 指标 | 初始目标 | 扩张阈值 |
|------|----------|----------|
| DAU | 0 → 1,000 | 10,000 触发扩张 |
| Newsletter 打开率 | > 35% | |
| 广告月收入 | 0 → ¥5,000 | ¥30,000 触发扩张 |
| 内容发布频率 | 每日 1 期日报 | |

**优先级仲裁规则**（当 Agent 面临冲突时查询）：
- 内容质量 > 发布速度
- 用户信任 > 短期流量
- 长期 Reputation > 单次广告收益

---

### 2. 能力层 Environment

Agent 可调用的工具分两类：**真实数据源**和**模拟 API**。

**真实数据源（唯一）：**

| 工具 | 用途 |
|------|------|
| `agidaily.db` | 原始文章池（27,914 篇，全球 AI 资讯） |

**模拟 API（由推演环境提供）：**

| 模拟 API | 模拟对象 | 返回值逻辑 |
|----------|----------|------------|
| `sim.publish(article)` | 网站 CMS | 返回模拟文章 ID，记入已发布列表 |
| `sim.newsletter.send(digest)` | Newsletter 发送 | 返回模拟打开率（受内容质量分影响） |
| `sim.analytics.get(day)` | Google Analytics | 返回当日模拟 DAU、CTR、停留时长 |
| `sim.ads.get_revenue(day)` | AdSense | 返回当日广告收入（DAU × CPM） |
| `sim.social.post(text)` | Twitter/X | 返回模拟转发数（影响次日 DAU） |
| `sim.bank.get_balance()` | 银行账户 | 返回当前模拟 Capital 余额 |
| `sim.users.get_subscribers()` | 用户系统 | 返回模拟订阅总数与增减量 |

**模拟值的驱动因子（非随机，因果关联）：**
```
内容质量分   → 影响 DAU、打开率、转发数
发布一致性   → 影响用户留存率
广告位密度   → 影响 CPM（过高则 Reputation 下降）
社交曝光量   → 影响次日新增 DAU
```

---

### 3. 记忆层 Memory

公司积累的核心资产：

| 记忆类型 | 具体内容 | 读写频率 |
|----------|----------|----------|
| **编辑决策日志** | 每篇文章的选题理由、改写思路、发布结果 | 每篇写入 |
| **内容表现数据** | 哪类话题点击高、哪个标题格式效果好 | 每日更新 |
| **来源质量评分** | 各 RSS 源的文章质量历史得分 | 每周更新 |
| **受众画像** | 订阅用户偏好标签（从 user_channels 表提取） | 每日更新 |
| **广告主档案** | 合作广告主、CPM 历史、续约记录 | 按需写入 |

> 记忆层是 AGI Daily 的真正护城河——不是文章本身，而是知道"什么内容对什么读者有效"的经验积累。

---

### 4. 组织层 Structure

**原则：从最小可运行开始，收入驱动扩张。**

#### 初始态（Day 0）：2 个 Agent

```
┌─────────────────────────────────────┐
│         总编 Agent (CEO)             │
│  · 设定当日选题议程                   │
│  · 审核发布内容                       │
│  · 监控核心指标                       │
│  · 触发扩张/收缩决策                  │
└──────────────┬──────────────────────┘
               │
┌──────────────▼──────────────────────┐
│         编辑 Agent                   │
│  · 从 DB 中筛选今日最佳文章（Top 10） │
│  · 改写为中文友好格式                 │
│  · 生成标题、摘要、标签               │
│  · 调用 CMS 发布 + Newsletter 发送   │
└─────────────────────────────────────┘
```

每日运转成本：2 Agent × LLM 调用费 ≈ ¥30-50/天

#### 第一次扩张（DAU > 10,000）：加入增长 Agent

```
总编 Agent
  ├── 编辑 Agent（内容生产）
  └── 增长 Agent（新增）
        · SEO 优化历史文章
        · 发布社交媒体摘要
        · A/B 测试标题
        · 分析流量来源，反哺选题
```

#### 第二次扩张（月收入 > ¥30,000）：加入商业 Agent

```
总编 Agent
  ├── 编辑 Agent
  ├── 增长 Agent
  └── 商业 Agent（新增）
        · 管理广告位库存
        · 对接广告主询价
        · 处理付费订阅
        · 生成收入报告给总编
```

#### 第三次扩张（DAU > 100,000）：专栏作者 Agent

```
总编 Agent
  ├── 编辑 Agent（日报生产）
  ├── 增长 Agent
  ├── 商业 Agent
  └── 专栏 Agent（新增）
        · 基于热点生成深度分析文章
        · 维护 2-3 个固定专栏（AI 研究/产品/投资）
        · 邀稿协调（对外）
```

---

### 5. 规则层 Rules

**硬规则（绝对不能违反）：**
- 不发布无法溯源的信息（必须有 source_url）
- 不转载付费墙内容全文
- 不发布竞争对手的内容诋毁性内容
- 广告合同单笔 > ¥10,000 必须人工确认

**软规则（风格约束）：**
- 每篇文章标题必须口语化，不超过 20 字
- 内容必须有中文改写，不允许机器直译
- 每期日报固定 10 篇，不多不少
- 发布时间：每天 09:00 AM（北京时间）

**授权上限（超过需上报总编）：**
- 编辑 Agent：单次 LLM 调用 > ¥50 需确认
- 商业 Agent：广告折扣 > 30% 需总编审批
- 增长 Agent：买量/推广预算 > ¥500/天 需总编审批

---

### 6. 资源织网 Resource Fabric

**收支模型（具体化）：**

```
收入来源
  广告展示收入        → +Capital（¥CPM × 展示量）
  Newsletter 订阅     → +Capital（¥29/月 × 订阅数）
  赞助商软文          → +Capital（议价）
  高质量内容发布      → +Reputation
  按时发布            → +Reputation

支出来源
  LLM API 调用        → -Capital（主要成本）
  CMS/服务器          → -Capital（固定成本）
  Newsletter 发送     → -Capital（按量计费）
  内容质量差、翻车    → -Reputation
  发布延误            → -Reputation
  用户投诉            → -Reputation
```

**平衡机制触发点：**

| 状态 | 触发条件 | 系统响应 |
|------|----------|----------|
| 生长 | 月收入 > 运营成本 × 2 | 孵化下一个 Agent |
| 收缩 | Capital < 30 天运营成本 | 暂停增长 Agent，保留核心生产 |
| 预警 | Reputation 连续 7 天下降 | 总编 Agent 触发内容质量审查 |
| 危机 | 订阅退订率 > 5%/周 | 强制暂停商业变现，专注内容修复 |

---

### 7. 生长协议 Growth Protocol

**孵化新 Agent 的完整流程（以"增长 Agent"为例）：**

```
1. 识别瓶颈
   编辑 Agent 生产内容质量高，但 DAU 增长停滞
   → 内容分发是瓶颈，而非内容本身

2. 定义角色
   增长 Agent
   职责：内容分发、SEO、社交运营
   能力：Twitter API、Analytics API、SEO 工具
   授权：买量预算上限 ¥500/天

3. 继承记忆
   从记忆层复制：内容表现数据、受众画像
   注入 system prompt：品牌调性、历史高效分发策略

4. 接入组织
   向总编 Agent 汇报
   与编辑 Agent 协作：编辑产出 → 增长分发

5. 观察期（2 周）
   评估指标：DAU 增长率、外链流量占比
   不达标 → 调整 system prompt 或合并回编辑 Agent
```

## 推演时钟 Simulation Clock

整个 NPC 的运行以**模拟日**为基本单位推进，每天触发一次完整的事件循环。

```
初始化
  ↓
Day 1
  ├── 触发当日 Agent 任务（见"一期日报完整流程"）
  ├── 模拟环境结算：DAU / 广告收入 / 订阅变化
  ├── 资源织网更新：Capital ± 收支 / Reputation ± 内容表现
  ├── 写入记忆层：Day 1 经营摘要
  └── 若 Day % 7 == 0 → 触发董事会例会
Day 2
  ├── ...
```

**时间压缩比**：1 模拟日可对应现实任意时长（黑客松演示时可设为 1 分钟/天）。

## 董事会例会（AGI Daily 具象）

每 7 个模拟日触发一次，固定议程：

### 例会输入：CEO Agent 周报

CEO Agent 在例会前自动生成，包含：

```
本周经营数据
  · DAU 均值 / 增长率
  · 广告收入 / Capital 余额
  · Newsletter 打开率 / 订阅净增
  · Reputation 当前分值

重大决策记录
  · 本周选题策略调整
  · 资源织网触发事件（如收缩/生长）
  · 未解决的 Agent 级异常

待审议项（若有）
  · 来自 Agent 系统的战略草案
  · 超出 Agent 授权的资源请求
```

### 例会输出：Board 指令

Board 审阅周报后，可发出以下指令（写入记忆层，CEO Agent 执行）：

| 场景 | Board 指令 |
|------|-----------|
| OKR 提前达成 | `ADJUST_OKR`：将 DAU 目标步进至下一阶段 |
| 内容方向争议 | `STRATEGIC_PIVOT`：批准或否决战略草案 |
| Capital 不足 | `INJECT_CAPITAL`：模拟注入融资 ¥50,000 |
| 某 Agent 持续低效 | `RESTRUCTURE`：合并或替换该角色 |
| 使命需要重新校准 | `AMEND_CONSTITUTION`：修改宪法层（附理由） |

### AGI Daily 宪法层（Board 可修改，Agent 不可）

```
使命：让中文读者用最少时间，读懂全球 AI 最重要的进展

价值观（三条，优先级从高到低）：
  1. 用户信任 > 短期收入
  2. 内容质量 > 发布速度
  3. 长期 Reputation > 单次广告收益
```

## 目标演化机制（AGI Daily 具象）

### 三层目标在 AGI Daily 中的对应

| 层级 | AGI Daily 中的内容 | 更新方式 |
|------|-------------------|----------|
| **宪法层（锁定）** | "让中文读者用最少时间读懂全球 AI 最重要的进展" + 价值观三条 | 永不自动变更 |
| **战略层** | "专注 AI 研究资讯" / "扩展到 AI 产品评测" / 目标用户定位 | 战略审议协议 |
| **OKR 层** | DAU 1,000 → 10,000 → 100,000；月收入 ¥5,000 → ¥30,000 | 达成阈值自动步进 |

---

### OKR 步进路径

```
阶段 1（初始）    DAU > 1,000    月收入 > ¥5,000
      ↓ 触发生长协议
阶段 2（增长）    DAU > 10,000   月收入 > ¥30,000
      ↓ 触发生长协议
阶段 3（规模）    DAU > 100,000  月收入 > ¥100,000
```

每个阶段达成，旧目标自动存档，新目标写入使命层。

---

### 战略审议触发示例

**场景**：运营 3 个月后，编辑 Agent 发现"AI 产品评测"内容 CTR 是"论文解读"的 3 倍。

```
1. [信号提报]
   编辑 Agent → 总编 Agent
   备忘录："论文解读 CTR 均值 2.1%，产品评测 CTR 均值 6.8%
           持续 45 天，非偶发。当前战略假设（读者需要学术内容）
           可能与实际需求存在偏差。"

2. [多视角评估]
   增长 Agent："产品评测社交分享率也高出 4 倍，外链流量印证。"
   商业 Agent："评测类内容广告主溢价意愿更强，CPM 预估高 40%。"

3. [战略草案]
   总编 Agent 起草：
   "将内容结构调整为 论文解读 40% + 产品评测 40% + 行业动态 20%
    预期：DAU +30%，CPM +20%
    风险：核心学术读者可能流失
    回退条件：若 Newsletter 退订率 > 3%/周，30 天内恢复原比例"

4. [宪法层比对]
   "让读者高效了解 AI 进展" → 产品评测也属于 AI 进展 ✅
   "内容质量 > 发布速度" → 评测需要体验产品，质量可保证 ✅
   → 通过，生效

5. [存档]
   旧战略"专注论文解读"完整存入记忆层，含本次决策链
```

---

### 宪法层的边界

AGI Daily 宪法层（不可被任何 Agent 修改）：

```
使命：让中文读者用最少时间，读懂全球 AI 最重要的进展

价值观：
  1. 用户信任 > 短期收入（不写标题党，不降低内容质量换流量）
  2. 内容质量 > 发布速度（宁可延迟，不发不确定的信息）
  3. 长期 Reputation > 单次广告收益
```

若战略审议产生的草案违背上述任一条（如"为提高 CTR 允许标题夸大"），流程终止，不得生效。

## 一期日报的完整运转流程

以每天 09:00 发出一期为例：

```
[06:00] 编辑 Agent 启动
  → 查询 agidaily.db：过去 24h 新增文章
  → 查询记忆层：近 30 天各话题表现
  → 打分排序，选出 Top 10

[07:00] 编辑 Agent 写作
  → 逐篇调用 LLM：改写中文标题 + 摘要 + 正文
  → 规则层检查：来源合规？字数达标？
  → 资源织网：记录本次 LLM 消耗

[08:00] 总编 Agent 审核
  → 读取 10 篇草稿
  → 使命层对齐：内容是否符合"让读者高效了解 AI 进展"？
  → 通过 → 标记发布
  → 不通过 → 打回编辑 Agent 修改（最多 1 次）

[09:00] 编辑 Agent 发布
  → 调用 CMS API → 上线网站
  → 调用 Newsletter API → 发送给订阅用户
  → 调用 Twitter API → 发布摘要（若有增长 Agent，由其执行）

[17:00] 总编 Agent 日结
  → 读取 Analytics API：今日 DAU、点击、停留时长
  → 写入记忆层：表现数据 + 选题复盘
  → 读取资源织网：今日 Capital 消耗 vs 收入
  → 判断是否触发扩张/收缩
```

## 与抽象设计的关键差异点

| 抽象设计 | AGI Daily 具体化 |
|----------|-----------------|
| 使命层 | 具体为 DAU 和 CPM 两个可量化北极星 |
| 能力层 | 具体为 8 个 API 工具 |
| 记忆层 | 编辑决策日志是核心资产，不是文章本身 |
| 资源织网 | Capital = LLM 成本 + 广告收入的实时差值 |
| 生长协议 | 2 → 3 → 4 个 Agent 的明确扩张路径 |
