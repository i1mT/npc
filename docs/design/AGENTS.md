# 设计文档

本目录保存 AGI Daily NPC 的界面、交互体验、组件规范和视觉系统上下文。

- `design-system.md`：当前视觉 token、组件风格和布局约束。

## 产品界面基调

AGI Daily NPC 有两个互补界面：

- 前台日报：偏杂志、报纸、内容消费体验，强调“这家公司今天对外产出了什么”。
- 后台控制台：偏运营系统、可解释性工具，强调“结果为什么发生、由谁导致、影响了哪些层”。

设计应避免营销式 landing page，优先构建可操作、可扫描、可追踪的真实工作界面。

## 关键交互范型

- Day 选择器：所有回放、事件、七层资产都围绕模拟日切换。
- 三件套：七层页面统一呈现快照、变更、来源事件。
- 事件穿透：工作事件可查看影响面，变更应能回到来源事件。
- 资源告警：Capital、Reputation、DAU、Subscribers 等指标需要清楚表达趋势和风险。
- 董事会日：董事会等待、输入指令、恢复运行应成为明显状态。

## 组件使用原则

1. 后台页优先复用 `AdminShell`、`Panel`、`HumanData`、`WorkEventCard`。
2. 图标优先使用 `lucide-react`。
3. 卡片用于信息单元，不把页面大区块过度卡片化。
4. 工具、控制、刷新、推进等动作按钮使用图标加短文本。
5. 事件、规则、资源、七层资产要保留高密度信息，避免为了装饰牺牲可读性。
